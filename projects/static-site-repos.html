static website repos

Private source, public build artifact

The website repos do not need Python migration. Their optimum browser shape is static HTML, CSS, and JavaScript, with the final public build separated from the private working source.

Open deploy build Open account site repo

Reusable build idea

Publish only final static files

name: Build static site
on:
  push:
    branches: [main]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Copy browser files
        run: mkdir -p dist && cp -R index.html styles.css projects dist/
      - uses: actions/upload-pages-artifact@v3
        with:
          path: dist
